# 启航平台 UI/UX 全面优化 - 深度审计版验证清单 v2.0

## 一、🔴 致命级问题验收（必须全部通过，否则项目不可上线）

### 1.1 颜色系统与无障碍 [Task 1]
- [ ] axe-core自动化扫描结果：0个critical violation，0个severe violation
- [ ] Chrome DevTools Lighthouse Accessibility评分 ≥ 95分
- [ ] 所有正文文本颜色对比度 ≥ 4.5:1（WCAG AA标准）
- [ ] 所有大号文本（标题）对比度 ≥ 3:1
- [ ] Chrome/Firefox/Safari/Edge四个浏览器颜色显示一致
- [ ] Windows高对比度模式下可正常使用所有功能
- [ ] 色盲模拟测试通过（protanopia/deuteropia/tritanopia三种模式）
- [ ] 所有interactive元素有focus-visible样式（键盘Tab可见）
- [ ] backdrop-blur在不支持的浏览器上降级为实色背景（无白屏）
- [ ] 半透明色（bg-white/XX）在深色背景上文字依然可读

**验证工具**: axe DevTools、Lighthouse、Chrome DevTools对比度检查器、Colour Contrast Analyzer

### 1.2 悬浮客服按钮 [Task 2]
- [ ] 桌面端拖拽结束后按钮自动吸附到左边缘或右边缘（距离16px）
- [ ] 移动端拖拽区域限制：不遮挡底部导航栏（距离底部≥80px）
- [ ] 拖拽过程中按钮透明度降至0.8，松手后恢复1.0
- [ ] 按钮无法被拖出屏幕可视区域（上下左右边界检测）

- [ ] 触摸设备上拖拽流畅，无延迟或卡顿
- [ ] 双击可快速打开聊天页（如启用此功能）

**验证方法**: 手动测试 + 录屏回放

### 1.3 聊天系统核心体验 [Task 3]
#### 用户端（Chat.tsx）
- [ ] 未登录用户可以看到FAQ常见问题列表（至少10条）
- [ ] 未登录用户可以进入游客模式发送消息（限制3条）
- [ ] 游客模式引导登录的转化流程顺畅
- [ ] 输入框工具栏包含：表情、附件、图片按钮（至少2个）
- [ ] 发送按钮尺寸 ≥ 48×48px（移动端触摸友好）
- [ ] Enter键发送消息，Shift+Enter换行（与当前Ctrl+Enter统一）
- [ ] 消息状态流转正确：
  - ⏳ 发送中显示三点跳动动画
  - ✓ 发送成功后显示单勾（已送达）
  - ✓✓ 对方查看后显示双勾（已读）
  - ❌ 发送失败显示红色感叹号，点击可重发
- [ ] 断网环境下发送消息失败，网络恢复后自动重发或提示重试
- [ ] 乐观更新正常：消息先显示在列表，后台失败后回滚并提示
- [ ] 移动端会话列表/聊天视图切换流畅

#### 管理员端（ChatManage.tsx）[Task 4]
- [ ] 管理员可创建自定义快捷回复模板（至少10条预设模板）
- [ ] 快捷回复支持一键插入到输入框
- [ ] 勾选多个会话后可批量关闭/标记已读
- [ ] 消息右键菜单包含：复制、撤回、标记已读
- [ ] 会话列表显示等待时长，超过5分钟高亮显示
- [ ] 统计面板展示：平均响应时间、解决率、今日处理数

- [ ] 所有卡片点击时有缩放反馈（active:scale-[0.98]）

- [ ] 键盘Enter或Space键可触发卡片点击事件
- [ ] 卡片右上角有箭头图标（→）暗示可点击（可选）
- [ ] 点击卡片后有涟漪效果（Ripple Animation）（可选）

## 四、综合质量门禁 [Task 15]

### 4.1 性能基准
- [ ] Lighthouse Performance ≥ 90
- [ ] Lighthouse Accessibility ≥ 95
- [ ] Lighthouse Best Practices ≥ 90
- [ ] Lighthouse SEO ≥ 80（如需SEO）
- [ ] 首屏JS Bundle Size (gzipped) ≤ 200KB
- [ ] CLS (Cumulative Layout Shift) ≤ 0.1

### 4.2 兼容性测试
- [ ] Chrome最新版（稳定版）全部功能正常
- [ ] Firefox最新版全部功能正常
- [ ] Safari最新版（macOS/iOS）全部功能正常
- [ ] Edge最新版全部功能正常
- [ ] 以上浏览器控制台无JavaScript错误
- [ ] 以上浏览器无网络请求失败（404/500等）

### 4.3 用户验收测试（UAT）
- [ ] 5名目标用户参与测试
- [ ] 核心任务成功率 ≥ 90%
  - 任务1: 注册/登录并完善个人信息
  - 任务2: 搜索岗位并查看详情
  - 任务3: 浏览导师并预约咨询
  - 任务4: 使用客服聊天功能

- [ ] 无硬编码的API密钥或密码（环境变量检查）
- [ ] XSS防护到位（用户输入转义）

## 五、细节检查清单（精选Top 50，参考Appendix B）

### 导航与信息架构
- [ ] Logo点击回到首页
- [ ] 面包屑导航（Breadcrumb）在二级以上页面显示
- [ ] 404页面有引导回家按钮和搜索框
- [ ] 当前页面在导航菜单中高亮显示
- [ ] "回到顶部"按钮在滚动超过一屏后出现

### 表单与输入
- [ ] 密码输入框有显示/隐藏切换
- [ ] 下拉选项超过10个时支持搜索筛选
- [ ] 表单验证错误信息具体（"请输入正确的手机号"而非"格式错误"）
- [ ] 必填字段有明确标识（*号或红色边框）
- [ ] 表单提交按钮有loading状态防重复提交
- [ ] 提交成功后有Toast提示（"保存成功"）

### 反馈与状态
- [ ] 删除操作有二次确认对话框
- [ ] 网络断开时有友好的离线提示（非alert弹窗）
- [ ] 图片加载失败显示主题色占位符（非空白或破损图标）
- [ ] 空状态有插图和引导操作（"暂无数据，去添加吧→"）
- [ ] 长列表支持虚拟滚动或分页（性能保障）
- [ ] 复制操作给予视觉反馈（Toast"已复制到剪贴板"）

### 无障碍（a11y）
- [ ] 所有图片有alt文本（装饰性图片alt=""）
- [ ] 链接有描述性的文本（非"点击这里"而是"查看岗位详情"）
- [ ] 颜色不是唯一的信息传达方式（图标+文字）
- [ ] 跳转到主内容的链接（Skip to main content）
- [ ] 页面标题（<title>）准确描述当前页面内容

### 性能与监控
- [ ] 控制台无console.error或console.warn（开发阶段warning可接受）
- [ ] 内存泄漏检查：切换页面20次后内存无明显增长
- [ ] 大图使用WebP格式，提供JPEG fallback
- [ ] 字体使用system font栈或可变字体（减少加载时间）
- [ ] 第三方脚本（统计/监控）异步加载，不阻塞渲染
