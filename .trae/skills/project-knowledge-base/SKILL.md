## 解决方案库

### 技术问题及解决方法

| 问题 | 解决方法 | 应用场景 |
|------|----------|----------|
| 前端服务启动失败 | 使用 PowerShell 语法：`Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; cd frontend; npm run dev` | 开发环境启动 |
| 端口占用 | Vite 自动切换端口（5173 → 5174 → 5175） | 多服务并行开发 |
| ErrorState compact 变体 | 已存在但可增强：添加 size、inline、图标位置等属性 | 不同场景的错误提示 |
| 图片加载优化 | 使用 Skeleton 骨架屏 + 品牌色占位 | 提升用户体验 |
| 配置管理 | 参考留学板块的 JSON 配置 + 可视化编辑页面 | 无代码内容管理 |

### 设计问题及解决方法

| 问题 | 解决方法 | 应用场景 |
|------|----------|----------|
| 图片资源不统一 | 建立统一规范（企业Logo 80x80、导师头像 300x300等） | 全站视觉一致性 |
| 交互反馈不足 | 实现 Ripple 点击效果、Toast 提示、微动效 | 提升操作体验 |
| 新手引导生硬 | 重构为气泡式引导，支持分场景触发 | 降低用户学习成本 |
| 无障碍支持缺失 | 添加 ARIA 标签、键盘导航、焦点样式 | 符合 WCAG 标准 |

## 项目构建指南

### 前端项目构建流程

1. **环境搭建**
   - Node.js 18.x 或 20.x
   - npm 安装依赖：`cd frontend && npm install`
   - 启动开发服务：`npm run dev`

2. **代码规范**
   - TypeScript + ESLint
   - Tailwind CSS 样式
   - Framer Motion 动画
   - Lucide React 图标

3. **组件开发**
   - 原子化组件设计
   - 四态组件库（Loading/Success/Error/Empty）
   - 响应式设计
   - 无障碍支持

4. **配置管理**
   - JSON 配置文件
   - 可视化配置页面
   - 配置修改日志

### 后端项目构建流程

1. **环境搭建**
   - Express + MySQL 8.0
   - JWT 认证
   - bcryptjs 密码加密

2. **API 设计**
   - RESTful API 规范
   - 角色权限控制
   - 错误处理

3. **数据库设计**
   - 16 张核心表
   - 种子数据
   - 索引优化

## 常见问题快速参考

### 开发环境

- **npm install 失败**：检查网络连接，清理 node_modules
- **端口占用**：使用不同端口或关闭占用端口的进程
- **TypeScript 错误**：检查类型定义，运行 `npm run lint`

### 部署问题

- **构建失败**：检查依赖版本，运行 `npm run build`
- **API 调用失败**：检查环境变量，API 地址配置
- **数据库连接**：检查数据库配置，网络连接

### 性能问题

- **页面加载慢**：优化图片，代码分割
- **API 响应慢**：数据库索引，缓存策略
- **内存泄漏**：检查事件监听器，清理定时器
