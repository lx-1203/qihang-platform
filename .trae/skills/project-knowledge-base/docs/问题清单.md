# 问题清单

## 用户提出的问题

### 1. 安全与可靠性问题
- **JWT_SECRET硬编码默认值**：生产环境易泄露，存在安全风险
- **Refresh Token黑名单使用内存Set**：多实例部署时无法共享，影响认证可靠性
- **开发模式跳过权限校验的后门**：存在安全隐患
- **接口限流使用内存Map**：多实例部署无法统一限流，易被攻击
- **ENCRYPTION_KEY示例为空**：生产环境可能未配置，影响数据加密
- **缺少文件上传病毒扫描机制**：存在安全风险
- **数据库连接池限制为10**：高并发场景可能成为瓶颈
- **缺少健康检查深度检测**：无法及时发现系统依赖问题

### 2. 用户体验问题
- **忘记密码功能是占位符**：用户无法找回密码
- **简历上传只能用链接**：学生无法方便上传简历文件
- **首页搜索只跳岗位**：不支持课程/导师搜索，用户体验不一致
- **大部分页面未使用Skeleton组件**：加载时白屏，体验差
- **Token刷新失败未保存操作状态**：用户操作丢失，体验差
- **无邮箱验证机制**：账号安全和垃圾账号风险
- **无批量处理简历功能**：企业招聘效率低
- **AI人才推荐是占位符**：功能承诺未兑现
- **无路由懒加载预取机制**：首屏加载速度可优化
- **无消息红点全局提示**：用户易错过重要通知
- **无面试日程管理**：企业无法安排面试时间
- **无深色模式**：长时间使用眼睛疲劳
- **无障碍支持不完善**：特殊用户体验差
- **无离线缓存（PWA）**：网络差时体验差
- **投递成功后无后续引导**：用户不知道下一步做什么
- **收藏无批量管理**：批量删除等操作效率低
- **简历池拖拽排序是占位符**：功能承诺未兑现
- **无快捷键支持**：操作效率可提升
- **无操作撤销机制**：误操作无法恢复
- **无智能搜索联想**：首页搜索体验不一致
- **无数据导出功能**：企业无法导出招聘数据
- **无自定义主题色**：个性化不足
- **无性能监控上报**：无法收集真实用户体验数据
- **无操作历史记录**：无法回溯用户操作
- **无分享功能**：无法分享职位/课程给朋友
- **无多语言切换入口**：虽然有i18n支持，但无切换入口
- **无手势支持（移动端）**：移动端体验可优化
- **无虚拟列表（大数据量）**：大数据场景性能可优化
- **无图片懒加载占位优化**：图片加载体验可提升
- **无面包屑导航**：除职位详情外，其他页面导航路径不清晰
- **无快速返回顶部按钮**：长页面浏览体验可优化

### 3. 项目管理问题
- **缺少测试计划、测试用例、验收标准文档**：质量保障不足
- **里程碑日期与当前日期接近**：时间紧张，可能影响质量
- **四期开发计划缺少WBS分解**：缺少关键路径分析和资源分配
- **缺少CI/CD流水线配置**：部署效率和质量保障不足

### 4. 功能需求
- **学生端功能增强**：
  - 个人画像与智能推荐
  - 简历管理中心
  - 校招日历与日程管理
- **企业端功能增强**：
  - 人才智能匹配
  - 招聘数据分析
- **导师端功能增强**：
  - 辅导内容库建设
  - 学生管理
- **管理后台功能增强**：
  - 配置化管理系统
  - 主题配置
  - 公告发布系统

## 技术问题

### 1. 前端开发
- **前端服务启动**：PowerShell执行权限问题
- **端口占用**：多服务并行开发时的端口冲突
- **TypeScript类型定义**：组件类型声明和类型检查
- **性能优化**：图片加载、代码分割、预加载

### 2. 后端开发
- **数据库连接**：MySQL连接配置和优化
- **API设计**：RESTful API规范和权限控制
- **安全性**：JWT认证、密码加密、SQL注入防护

### 3. 部署问题
- **构建失败**：依赖版本冲突
- **环境配置**：不同环境的配置管理
- **性能监控**：生产环境的性能监控和日志

## 设计问题

### 1. 视觉设计
- **图片资源规范**：统一图片尺寸和格式
- **品牌视觉统一**：保持品牌风格一致性
- **响应式设计**：适配不同设备和屏幕尺寸

### 2. 交互设计
- **操作反馈**：点击效果、加载状态、错误提示
- **新手引导**：分场景引导和用户教育
- **无障碍设计**：键盘导航、屏幕阅读器支持

### 1. 开发流程
- **版本控制**：Git分支管理和代码合并
- **CI/CD**：自动化测试和部署
- **文档管理**：代码注释和API文档

### 2. 团队协作
- **代码规范**：统一的代码风格和命名规范
- **沟通机制**：团队沟通和问题解决流程
