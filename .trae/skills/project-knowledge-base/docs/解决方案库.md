# 解决方案库

## 技术问题及解决方法

### 前端开发

| 问题 | 解决方法 | 应用场景 |
|------|----------|----------|
| **前端服务启动失败** | 使用 PowerShell 语法：`Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; cd frontend; npm run dev` | 开发环境启动 |
| **端口占用** | Vite 自动切换端口（5173 → 5174 → 5175），或手动指定端口：`npm run dev -- --port 3000` | 多服务并行开发 |
| **TypeScript 类型错误** | 运行 `npm run lint` 检查类型错误，使用 `tsc --noEmit` 验证类型 | 代码质量检查 |
| **图片加载优化** | 使用 Skeleton 骨架屏 + 品牌色占位，实现懒加载 | 提升用户体验 |
| **ErrorState compact 变体** | 已存在但可增强：添加 size、inline、图标位置等属性 | 不同场景的错误提示 |
| **配置管理** | 参考留学板块的 JSON 配置 + 可视化编辑页面 | 无代码内容管理 |

### 后端开发

| 问题 | 解决方法 | 应用场景 |
|------|----------|----------|
| **数据库连接失败** | 检查 MySQL 服务是否启动，验证连接配置 | 开发和生产环境 |
| **API 权限问题** | 确保 JWT token 正确验证，检查中间件配置 | 接口安全 |
| **密码加密** | 使用 bcryptjs 进行密码加密，设置适当的哈希强度 | 用户认证 |
| **SQL 注入防护** | 使用参数化查询，避免直接拼接 SQL 语句 | 数据安全 |

## 设计问题及解决方法

### 视觉设计

| 问题 | 解决方法 | 应用场景 |
|------|----------|----------|
| **图片资源不统一** | 建立统一规范：<br>- 企业Logo：80x80px<br>- 导师头像：300x300px<br>- 课程封面：400x225px<br>- 首页Banner：1920x600px | 全站视觉一致性 |
| **品牌视觉锤** | 创建 Logo + 渐变背景组件，统一品牌识别 | 品牌建设 |
| **响应式设计** | 使用 Tailwind CSS 的响应式类，确保在不同设备上的显示效果 | 多设备适配 |

### 交互设计

| 问题 | 解决方法 | 应用场景 |
|------|----------|----------|
| **交互反馈不足** | 实现 Ripple 点击效果、Toast 提示、微动效 | 提升操作体验 |
| **新手引导生硬** | 重构为气泡式引导，支持分场景触发（首次注册、新功能上线） | 降低用户学习成本 |

## 安全解决方案

| 问题 | 解决方法 | 技术要点 |
|------|----------|----------|
| **JWT_SECRET硬编码默认值** | 移除默认值，启动时强制校验环境变量，缺失则退出 | 配置管理 |
| **Refresh Token黑名单使用内存Set** | 迁移到MySQL表，SHA256哈希存储，定时清理过期Token | 后端架构 |
| **开发模式跳过权限校验的后门** | 改用`import.meta.env.DEV`，生产构建自动tree-shake | 安全加固 |
| **接口限流使用内存Map** | 标注局限性，后续迁移到Redis统一限流 | 系统架构 |
| **ENCRYPTION_KEY示例为空** | 加入必需环境变量，格式校验（64位十六进制） | 配置管理 |
| **缺少文件上传病毒扫描机制** | 添加Magic Bytes验证，防止文件伪造 | 安全加固 |
| **数据库连接池限制为10** | 改为环境变量可配置（默认20），根据服务器配置调整 | 性能优化 |
| **缺少健康检查深度检测** | 增加数据库连接状态+延迟检测 | 监控告警 |
| **XSS 攻击** | 输入验证，使用 DOMPurify 净化用户输入 | 前端安全 |
| **CSRF 攻击** | 使用 CSRF token，验证请求来源 | 后端安全 |
| **敏感数据泄露** | 使用环境变量存储敏感信息，加密传输 | 配置管理 |
