# 项目构建指南

## 前端项目构建流程

### 1. 环境搭建

#### 1.1 系统要求
- **操作系统**：Windows 10/11, macOS, Linux
- **Node.js**：18.x 或 20.x
- **npm**：8.x 或以上
- **Git**：2.30.x 或以上

#### 1.2 安装步骤
1. **克隆代码**
   ```bash
   git clone <repository-url>
   cd <project-directory>
   ```

2. **安装依赖**
   ```bash
   cd frontend
   npm install
   ```

3. **启动开发服务**
   ```bash
   # Windows PowerShell
   Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass; npm run dev
   
   # macOS/Linux
   npm run dev
   ```

4. **构建生产版本**
   ```bash
   npm run build
   ```

### 2. 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19.0.0 | 前端框架 |
| TypeScript | 5.2.0 | 类型系统 |
| Tailwind CSS | 4.0.0 | 样式框架 |
| Framer Motion | 10.16.0 | 动画库 |
| Lucide React | 0.294.0 | 图标库 |
| Zustand | 4.4.7 | 状态管理 |
| Vite | 5.0.0 | 构建工具 |

### 4. 组件开发规范

#### 4.1 组件命名
- **文件夹**：使用 PascalCase（如 `Button`、`ErrorState`）
- **组件文件**：使用 `.tsx` 扩展名
- **Props 类型**：使用 `interface` 定义，命名为 `ComponentProps`

#### 4.2 代码风格
- **缩进**：2 个空格
- **命名**：
  - 组件名：PascalCase
  - 变量和函数：camelCase
  - 常量：UPPER_SNAKE_CASE
- **注释**：为复杂逻辑添加注释

#### 4.3 组件设计原则
- **单一职责**：每个组件只负责一个功能
- **可复用性**：设计通用组件时考虑复用场景
- **可测试性**：组件逻辑清晰，便于测试
- **性能优化**：合理使用 `React.memo`、`useMemo`、`useCallback`

### 5. 状态管理

- **全局状态**：使用 Zustand 管理
- **局部状态**：使用 `useState`、`useReducer`
- **异步状态**：使用 `useQuery` 或自定义 hooks

### 6. API 调用

- **服务层**：在 `services/` 目录中定义 API 调用
- **错误处理**：统一的错误处理机制
- **请求拦截**：添加认证 token
- **响应拦截**：统一处理响应格式

## 后端项目构建流程

### 1. 环境搭建

#### 1.1 系统要求
- **Node.js**：18.x 或 20.x
- **MySQL**：8.0 或以上
- **Redis**（可选）：用于缓存

#### 1.2 安装步骤
1. **安装依赖**
   ```bash
   cd backend
   npm install
   ```

2. **配置环境变量**
   - 复制 `.env.example` 为 `.env`
   - 填写数据库连接信息和其他配置

3. **启动开发服务**
   ```bash
   npm run dev
   ```

4. **构建生产版本**
   ```bash
   npm run build
   ```

### 2. 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Express | 4.18.0 | 后端框架 |
| MySQL | 8.0 | 数据库 |
| Sequelize | 6.35.0 | ORM |
| JWT | 9.0.0 | 认证 |
| bcryptjs | 2.4.3 | 密码加密 |
| express-validator | 7.0.0 | 数据验证 |

### 3. 项目结构

```
backend/
├── src/
│   ├── config/        # 配置
│   ├── controllers/   # 控制器
│   ├── models/        # 数据模型
│   ├── routes/        # 路由
│   ├── middleware/    # 中间件
│   ├── services/      # 业务逻辑
│   ├── utils/         # 工具函数
│   ├── database/      # 数据库初始化
│   └── app.ts         # 应用入口
├── package.json       # 依赖配置
├── tsconfig.json      # TypeScript配置
└── .env.example       # 环境变量示例
```

### 4. 数据库设计

#### 4.1 核心表结构
- **users**：用户信息
- **students**：学生信息
- **enterprises**：企业信息
- **mentors**：导师信息
- **jobs**：职位信息
- **applications**：申请记录
- **courses**：课程信息
- **reviews**：评价信息

#### 4.2 数据库初始化
- **迁移**：使用 Sequelize 迁移管理数据库结构
- **种子数据**：初始化基础数据

### 5. API 设计规范

#### 5.1 RESTful API 规范
- **GET**：获取资源
- **POST**：创建资源
- **PUT**：更新资源
- **DELETE**：删除资源

#### 5.2 路由命名
- 版本前缀：`/api/v1`
- 资源命名：使用复数形式
- 示例：`/api/v1/students`, `/api/v1/jobs`

#### 5.3 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 6. 安全性

- **认证**：JWT token 验证
- **授权**：基于角色的权限控制
- **输入验证**：使用 express-validator
- **密码加密**：bcryptjs 哈希
- **CORS**：跨域资源共享配置

## 故障排查

### 1. 前端问题
- **构建失败**：检查依赖版本，运行 `npm run build`
- **页面空白**：检查控制台错误，网络请求
- **样式问题**：检查 Tailwind 类名，响应式设计

### 2. 后端问题
- **服务启动失败**：检查端口占用，环境变量
- **数据库连接失败**：检查数据库服务，连接配置
- **API 错误**：检查路由，控制器，数据库查询

### 3. 部署问题
- **环境变量**：确保生产环境变量正确配置
- **权限问题**：检查文件权限，服务启动权限
- **网络问题**：检查防火墙，网络连接
